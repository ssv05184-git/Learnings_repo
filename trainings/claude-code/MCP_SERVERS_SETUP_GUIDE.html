What is an MCP Server?

MCP = Model Context Protocol Definition: External tools that extend Claude Code's capabilities beyond file reading/writing. Examples:
  • Playwright - Control browsers, take screenshots, test UIs
  • GitHub - Read PRs, manage repos, integrate with Git workflows
  • Figma - Import design mockups into Claude's context
  • FRED - Access economic data via API
  • Node.js - Execute scripts, run tests
Why Use Them?
  • Automate UI testing
  • Integrate with external APIs
  • Get real-time data
  • Validate designs
  • Manage GitHub workflows

Installation Prerequisites

For All MCP Servers

1. Node.js 18+ (most MCP servers are Node-based)

   node --version  # Should be v18.0.0+

2. Valid API Keys (if the server requires authentication)

- GitHub: GitHub CLI or Personal Access Token

- Figma: Figma API Key

- FRED: FRED API Key

3. Claude Code installed 
   claude --version

Common MCP Servers

1. Playwright (Browser Automation)

Purpose: Automate UI testing, take screenshots, interact with web pages. Installation: 
# Exit Claude Code first
exit

# Add Playwright MCP server
claude mcp add playwright npx @playwright/mcp@latest

# Re-launch Claude Code
claude

# Verify
/mcp
Verify Connection: 
/mcp
# Should show: playwright (connected)
What You Can Do:
  • Launch browser: browser.launch()
  • Navigate to URL: browser.navigate("http://localhost:3000")
  • Take screenshot: browser.take_screenshot()
  • Fill form fields: browser.fill_field(selector, text)
  • Click elements: browser.click_element(selector)
  • Get page content: browser.get_page_content()
Example Prompts: 
claude

# Example 1: Visual verification
"Using Playwright, visit http://localhost:3000, take a screenshot, 
and verify the login button is centered and blue"

# Example 2: Screenshot analysis
"[paste screenshot from Win+Shift+S]
These fonts are hard to read. Make them bigger and bolder."

# Example 3: Automated testing
"Using Playwright, test the login flow:
1. Navigate to http://localhost:3000/login
2. Fill email field with test@example.com
3. Fill password field with testpass123
4. Click Login button
5. Verify redirect to dashboard
6. Take screenshot for verification"

First time Claude uses a Playwright tool:

3. Figma (Design Import)

Purpose: Import Figma mockups and build code from designs. Prerequisites:
  • Figma Desktop App installed
  • Figma account with design access
  • Dev Mode seat (paid) OR use free alternative (Framelink)

#### Option A: Official Figma MCP Server (Paid)

Requirements:
  • Dev or Full seat on Figma Pro+
  • Figma file in Dev Mode
Setup:

1. Open design in Figma Desktop

2. Menu ‚Üí Preferences ‚Üí Enable Dev Mode MCP Server

3. Confirmation message: "Running locally at http://127.0.0.1:3845/mcp"

4. Add to Claude Code:

exit  # Exit Claude Code

claude mcp add --transport http figma-dev-mode http://127.0.0.1:3845/mcp

claude  # Re-launch
Example Prompt: 
claude

"Using the Figma design I've shared, analyze the layout and build a React component that matches it.
Use Tailwind CSS for styling and ensure it's responsive."

#### Option B: Framelink MCP Server (Free)

Installation: 1. Get Figma API key from figma.com/developers

2. Add Framelink to Claude Code:

exit

claude mcp add "Framelink Figma MCP" \
  npx -y figma-developer-mcp \
  --figma-api-key=YOUR_API_KEY \
  --stdio

claude

3. Verify:

/mcp
Documentation:
  • Framelink Quickstart
  • Best Practices

4. FRED - Economic Data API

Purpose: Access Federal Reserve Economic Data for financial dashboards. Installation: 1. Get API key from fred.stlouisfed.org/docs/api

Add a New MCP Server

Generic syntax: 
exit  # Exit Claude Code first

claude mcp add <name> <command>

claude  # Re-launch
Examples: 
# Playwright
claude mcp add playwright npx @playwright/mcp@latest

# Via HTTP (Figma)
claude mcp add --transport http figma http://127.0.0.1:3845/mcp

# Custom command
claude mcp add "my-tool" "node /path/to/mcp-server.js"

Remove an MCP Server

exit

claude mcp remove playwright

claude

Permission Management

Why Permissions Matter

Claude Code asks for permission before using certain critical tools:

  • Taking screenshots
  • Writing to files outside your project
  • Running external commands
  • Accessing sensitive data

Troubleshooting MCP Servers

claude
/mcp

# If [Disconnected ‚úó]:

exit
npm list -g @playwright/mcp

# 2. Reinstall
npm uninstall -g @playwright/mcp
npm install -g @playwright/mcp

# 3. Re-add to Claude Code
claude mcp remove playwright
claude mcp add playwright npx @playwright/mcp@latest

# 4. Verify
claude
/mcp

node --version  # Ensure 18+

Best Practices

Do ‚úÖ

  • Use specific MCP tools for their intended purpose (Playwright for UI, GitHub for PRs)
  • Set permissions for tools you use frequently to avoid repeated prompts
  • Keep API keys secure - never commit them to git
  • Test with screenshots - take screenshots before and after changes

Don't ‚ùå

  • Share API keys - keep FRED, GitHub, Figma keys private
  • Run unvetted commands - always review what Claude Code will execute
  • Leave MCP servers running if you don't need them (exit Claude Code)
  • Change permissions globally unless you trust all code
