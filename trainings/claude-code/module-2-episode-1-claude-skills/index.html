üß≠ Quick Overview

The Big Picture:
Claude Skills are modular capabilities you can add to Claude Code. Unlike slash commands that you trigger manually, Claude Skills are model-invoked‚ÄîClaude automatically decides when to use them based on your request. Each skill lives in a folder with a skill.md file that describes what the skill does and when to use it.

What Problem Does This Solve? - Manual prompts are inconsistent and produce variable results
- Complex workflows require repeating the same instructions
- Developers need reusable, predictable automation without writing custom code

Where You'll Use This: - Automating code reviews and formatting
- Generating commit messages automatically
- Creating specialized workflows (PDF extraction, data cleanup)
- Building team-wide coding standards

üîë Core Concepts Explained

Concept 1: Model-Invoked Skills

Definition:
Claude automatically detects when a skill applies to your request and uses it without manual triggering.

Why It Matters:
This removes friction‚Äîyou don't need to remember to invoke a skill; Claude recognizes the context and acts appropriately.

When To Use:
For workflows that should run automatically based on context (code reviews, commit messages, formatting).

Concept 2: Skill.md Structure

Definition:
A markdown file that contains three essential components: name, description, and instructions.

Why It Matters:
The description tells Claude when to use the skill. Vague descriptions lead to missed opportunities; clear ones ensure proper activation.

When To Use:
Be specific about what the skill does and under what conditions Claude should invoke it.

Concept 3: Global vs Project-Level Skills

Definition:
Global skills live in ~/.claude/skills/ and apply everywhere. Project-level skills live in .claude/skills/ and apply only to that project.

Why It Matters:
Global skills enable consistency across all your work. Project-level skills allow customization for specific repositories.

When To Use:
Use global for universal standards (commit messages). Use project-level for domain-specific workflows.

How It Works: Step-by-Step

üìã Building Your First Skill: Commit Message Generator

Step 1: Create the Skill Folder

mkdir -p ~/.claude/skills/commit-helper
cd ~/.claude/skills/commit-helper

Step 2: Create skill.md

---
name: Commit Message Generator
description: Generates professional, clean Git commit messages automatically. Use this after code changes to create descriptive commit messages that follow best practices.
---

## Instructions

When Claude Code detects staged changes or when asked to generate a commit message:

1. Analyze the code changes and identify the core functionality updated
2. Categorize the change type (feat, fix, refactor, docs, style, test, chore)
3. Write a concise subject line (max 50 characters)
4. Provide a detailed body explaining the "why" behind changes
5. Format according to conventional commits standard

Example output:

feat: add user authentication module

  • Implement JWT-based authentication
  • Add password hashing with bcrypt
  • Create login/logout endpoints
  • Add session management

## When To Use This Skill

Claude automatically uses this skill when:
- Code changes are staged and ready for commit
- User explicitly asks for a commit message
- In CI/CD workflows requiring automated commit messages

Step 3: Activate the Skill

Simply restart Claude Code:

claude

Step 4: Test the Skill

claude "Generate a commit message for my staged changes"

Claude automatically recognizes and invokes the skill.
