Complete Feature Workflow

#!/bin/bash
# complete_workflow.sh

FEATURE_NAME="add-task-editing"
FEATURE_BRANCH="feature/$FEATURE_NAME"

echo "üöÄ Starting complete feature workflow for: $FEATURE_NAME"

# Phase 1: Planning
echo "üìã Phase 1: Planning"
claude "
Plan feature: $FEATURE_NAME
Requirements:
- Users can edit existing tasks
- Edit happens inline with save/cancel buttons
- Changes persist to database
- Optimistic updates for UX

Output a detailed technical specification including:
- Component architecture
- State management approach
- API endpoints needed
- Database schema changes
- Testing strategy
"

# Phase 2: Create Feature Branch
echo "üåø Phase 2: Creating feature branch"
git checkout -b "$FEATURE_BRANCH"
git push origin "$FEATURE_BRANCH"

# Phase 3: Implementation
echo "üíª Phase 3: Implementation"
claude "
Implement $FEATURE_NAME:

1. Create/update React components:
   - TaskEditForm component for inline editing
   - EditableTask wrapper component

2. Update state management (Redux/Context):
   - Actions for startEdit, saveEdit, cancelEdit
   - Selectors for editing state

3. Create API endpoints:
   - PATCH /tasks/:id for updates
   - Validation middleware

4. Update database:
   - Ensure updated_at timestamp
   - Version tracking for edits

5. Write comprehensive tests:
   - Unit tests for components
   - Integration tests for API
   - E2E tests for user flows

Commit changes with descriptive messages.
Ensure all tests pass.
"

# Phase 4: Code Quality
echo "üîç Phase 4: Quality Checks"
claude "
Review code quality:
1. Run linter: npm run lint
2. Run tests: npm run test
3. Check coverage: npm run coverage
4. Performance audit: npm run audit

If issues found:
- Fix linting errors
- Improve test coverage to >80%
- Address performance concerns
- Fix security vulnerabilities

Commit fixes if any.
"

# Phase 5: PR Creation
echo "üìù Phase 5: Creating Pull Request"
claude "
Create a comprehensive GitHub PR:

1. Title: Implement inline task editing

2. Description:
   - Problem statement
   - Solution overview
   - Breaking changes (if any)
   - Testing approach
   - Screenshots/demo

3. Labels: feature, enhancement

4. Reviewers: @team-lead, @senior-dev

5. Checklist:
   - [ ] Tests pass
   - [ ] Coverage > 80%
   - [ ] Documentation updated
   - [ ] No breaking changes
   - [ ] Performance reviewed

Use GitHub MCP to create the PR programmatically.
"

# Phase 6: Automated Review
echo "‚úÖ Phase 6: Automated Review"
claude "
Conduct thorough code review:

1. Security Check:
   - Input validation
   - SQL injection prevention
   - XSS prevention
   - Authentication/authorization

2. Performance:
   - Database query optimization
   - Component re-render prevention
   - Bundle size impact

3. Best Practices:
   - Code style compliance
   - Error handling
   - Logging appropriateness
   - Documentation

4. Test Coverage:
   - Edge cases covered
   - Error scenarios tested
   - Integration tests present

Provide detailed feedback. If issues found, fix them.
"

# Phase 7: Merge to Develop
echo "üîÄ Phase 7: Merging"
git checkout develop
git pull origin develop
git merge "$FEATURE_BRANCH"
git push origin develop

# Phase 8: Deploy to Staging
echo "üöÄ Phase 8: Deploy to Staging"
claude "
Trigger deployment to staging:
1. Via GitHub Actions or deployment script
2. Verify deployment succeeded
3. Run smoke tests against staging
4. Check error logs
"

# Phase 9: E2E Testing
echo "üß™ Phase 9: End-to-End Testing"
claude "
Run comprehensive E2E tests:

Test Scenarios:
1. Create task
2. Edit task with various inputs
3. Edit task with long text
4. Concurrent edits
5. Network error recovery
6. Undo edit (if applicable)
7. Delete edited task
8. Permission checks

Report results with screenshots/videos.
"

# Phase 10: Deploy to Production
echo "üéØ Phase 10: Deploy to Production"
claude "
Deploy to production:

Pre-deployment:
1. Final security scan
2. Performance check
3. Database migration validation
4. Backup verification

Deployment:
1. Create release tag
2. Deploy to production (blue-green or canary)
3. Monitor error rates
4. Check performance metrics

Post-deployment:
1. Verify feature works
2. Monitor logs for issues
3. Send deployment notification
4. Update documentation
"

# Phase 11: Documentation & Cleanup
echo "üìö Phase 11: Documentation"
claude "
Complete documentation:

1. Update README if needed
2. Add API documentation
3. Create user guide
4. Document database changes
5. Add release notes
6. Document any config changes
"

# Phase 12: Close and Cleanup
echo "üßπ Phase 12: Cleanup"
git branch -d "$FEATURE_BRANCH"
git push origin --delete "$FEATURE_BRANCH"

echo "‚úÖ Feature workflow complete!"

Real Example: React To-Do App

#!/bin/bash
# todo_app_example.sh

echo "üöÄ Building React To-Do App with Claude"

# Initial setup
claude "
Create a React To-Do application with:

Features:
1. View all tasks
2. Create new task
3. Mark task complete/incomplete
4. Edit task description (inline)
5. Delete task
6. Filter: All, Active, Completed
7. Task statistics (total, completed, remaining)

Tech Stack:
- React + TypeScript
- Redux for state
- Express.js backend
- PostgreSQL database
- Tailwind CSS styling

Please:
1. Create project structure
2. Set up all dependencies
3. Create database schema
4. Implement all components
5. Write comprehensive tests
6. Set up GitHub Actions CI/CD
7. Create deployment manifests
"

# Development workflow
for feature in \
  "task-editing" \
  "task-filtering" \
  "task-statistics"
do
  echo "Implementing: $feature"

  # Claude handles entire feature branch workflow
  claude "
  Implement feature: $feature
  - Create feature branch
  - Develop feature
  - Write tests
  - Create PR
  - Handle code review
  - Merge when ready
  "
done

echo "‚úÖ To-Do app complete and deployed!"
