üîë Core Concepts Explained

Concept 1: What Are Output Styles?

Definition:
Output styles are preset instructions that determine how Claude responds, including response length, detail level, and format.

Why It Matters:
Different tasks need different approaches. A learning objective requires step-by-step guidance; a coding task requires concise solutions.

When To Use:
Choose output styles based on your goal: learning, explaining, building, or reviewing.

Concept 2: The Three Built-in Styles

Definition:
Default (general coding), Explanatory (detailed understanding), and Learning (step-by-step guidance).

Why It Matters:
Each style optimizes for different outcomes‚Äîspeed, understanding, or skill development.

When To Use:
Match the style to your current objective.

Concept 3: Style Customization

Definition:
Output styles are just instructions; you can create custom ones tailored to your workflow.

Why It Matters:
Generic styles might not fit specialized domains (data analysis, DevOps, etc.).

When To Use:
When you have repeating workflows that need consistent, domain-specific responses.

‚öôÔ∏è Technical Deep Dive

The Three Built-in Styles

Style 1: Default

Purpose: Mainly used for coding tasks
Characteristics: - Provides code and short, clear explanations
- Focused on producing working solutions
- Best for: Writing features, fixing bugs, quick implementations

Example:

Input: "Add user authentication to my app"
Output: Code with brief explanation of key parts

Style 2: Explanatory

Purpose: Focuses on detailed explanations
Characteristics: - Provides extensive context and reasoning
- Explains why solutions work, not just how
- Best for: Learning new concepts, understanding design patterns, code reviews

Example:

Input: "Explain how this authentication works"
Output: Detailed breakdown of each component with conceptual discussion

Style 3: Learning

Purpose: Great for practicing and learning new concepts
Characteristics: - Provides small, guided tasks
- Encourages independent problem-solving
- Best for: Skill development, tutorials, workshops

Example:

Input: "Teach me React hooks"
Output: Step-by-step exercises with hints but not full solutions

üìã How to Change Output Styles

Using the /output-style Command

# Open the output style selector
/output-style

# You'll see options:
# 1. Default
# 2. Explanatory
# 3. Learning

Steps: 1. Type /output-style in Claude Code 2. Claude displays available styles 3. Select your desired style 4. Confirm selection 5. Claude switches behavior immediately

Example Workflow

claude> /output-style
? Select output style:
> Default
  Explanatory
  Learning

# Select "Learning"
‚úÖ Output style changed to Learning

claude> "Teach me async/await in JavaScript"
# Claude now responds with step-by-step learning approach

üí° Best Practices for Using Output Styles

Pro Tip 1: Match Style to Task

Task Best Style Reason
Building features Default Quick, efficient
Starting new project Explanatory Understand architecture
Learning framework Learning Progressive skill building
Code review Explanatory Detailed analysis
Bug fixing Default Direct solution

Pro Tip 2: Switch Styles Mid-Session

You can change styles mid-conversation to adapt to your needs:

# Start with Default for implementation
claude> /output-style Default

# Switch to Explanatory when confused
claude> /output-style Explanatory
"Why is this approach better?"

# Back to Default to continue coding
claude> /output-style Default

Pro Tip 3: Combine with Custom Output Styles

Start with built-in styles, then create custom ones for your specific needs.
