üîë Core Concepts Explained

Concept 1: Command Prefixes

Definition:

Claude Code uses special prefixes (/, !, @, &) to differentiate between commands, shell instructions, file paths, and background processes.

Why It Matters:

Understanding prefixes ensures Claude interprets your intent correctly ‚Äî whether you‚Äôre running a shell command or invoking a tool.

When To Use:

Use / for Claude Code commands, ! for shell execution, and @ for referencing files.

Concept 2: Context Management

Definition:

Context commands control the session‚Äôs memory footprint and relevance, including /clear, /compact, and /context.

Why It Matters:

Proper context management prevents Claude from becoming ‚Äúconfused‚Äù or exceeding token limits.

When To Use:

Before long sessions, after major topic changes, or when performance slows.

Concept 3: Agentic Task Control

Definition:

Commands like /tasks, /plan, and /fork enable Claude to act as an autonomous agent managing multiple concurrent operations.

Why It Matters:

You can delegate subtasks, track progress, and branch conversations for experimentation.

When To Use:

When orchestrating multi-step builds, testing scenarios, or refactoring sessions.
