File & Shell Commands

@ Symbol - File References

Precise referencing prevents mistakes: 
# Good - Claude knows exactly which files
claude "Add logging to @src/handlers.py but NOT @src/controllers.py"

# Also good - Multiple files
claude "Update @schema.py, @models.py, and @migrations.py to support composite keys"

# Avoid - Claude has to guess
claude "Add logging everywhere"

Advanced Patterns

Pattern: Safe Large Refactor

claude

# 1. Enable safety
Shift+Tab Shift+Tab

# 2. Reference specific files for precision
"Refactor @backend/app.py, @backend/models.py, and @backend/database.py 
to use a repository pattern instead of raw queries"

# 3. Review the detailed plan
# Press Y to approve

# 4. After completion, verify
!npm test              # Run tests
/cost                  # Check cost

Pattern: Continuous Development

# Continue your last session
claude -c

# Summarize previous work
/compact "Summarize the React components we built yesterday"

# Continue with next task
"Now add dark mode toggle to all components"

# Check your progress
/tasks

# When done, export your work
/export

Pattern: Compare Approaches (Forking)

claude

# Main approach: Use REST API
"Build a user authentication system using JWT tokens"

# Explore alternative: GraphQL
/fork
"Build the same auth system but using GraphQL instead"

# Go back to REST approach to compare
/resume "Build a user authentication system using JWT tokens"

# Choose the best approach based on comparison

Power User Tips

Tip 1: Create Reusable Commands

# Create command directory
mkdir -p .claude/commands

# Create add-tests.md
cat > .claude/commands/add-tests.md << 'EOF'
Write comprehensive unit tests for: $ARGUMENTS

Requirements:
- Test all functions and methods
- Include edge cases
- Use mocks for external dependencies
- Achieve >80% coverage
EOF

# Now use it
claude
/add-tests "for the payment processing module in @payments.py"

Tip 2: Use Hooks for Automation

In .claude/hooks.yaml:

hooks:
  after_tool_use:
    - name: "Format code after edits"
      command: "black . --quiet && npm run lint"
      
  after_response:
    - name: "Auto-commit if tests pass"
      command: |
        npm test && 
        git add -A && 
        git commit -m "Claude Code changes"
