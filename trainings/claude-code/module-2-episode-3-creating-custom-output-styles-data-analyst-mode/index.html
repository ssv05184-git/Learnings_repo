üîë Core Concepts Explained

Concept 1: Output Style Architecture

Definition:
Output styles are configuration templates that govern how Claude formats responses‚Äîstructuring data as tables, markdown, JSON, or custom formats.

Why It Matters:
Proper formatting reduces cognitive load and accelerates decision-making. A well-formatted table communicates insights faster than unstructured prose.

When To Use:
- When working with structured data (tables, time series)
- When generating reports that need consistent formatting
- When integrating Claude output into dashboards or BI tools

Concept 2: Data Analyst Mode

Definition:
A specialized output style optimized for data professionals, emphasizing summary statistics, tabular data, and visual insights over narrative explanations.

Why It Matters:
Data analysts work at speed; they need insights presented immediately without unnecessary context. Analyst mode strips unnecessary narrative and prioritizes patterns.

When To Use:
- Exploratory data analysis (EDA) workflows
- Statistical summaries and trend identification
- Business metrics reporting

Concept 3: Custom Output Formatters

Definition:
Functions or configurations that transform raw data into domain-specific formats (markdown tables, JSON schemas, LaTeX equations, etc.).

Why It Matters:
Enables seamless integration with downstream tools (BI platforms, markdown renderers, code generators).

When To Use:
When you need output automatically formatted for specific tools or reporting systems.

How Output Styles Work

Step 1: Style Configuration
Output styles are defined in .claude/config.json or via command-line flags:

  "formatting": {

    "charts": "ascii_art",
    "summary": true,
    "precision": 2
  }

üõ† Implementation Guide

Step 1: Enable Data Analyst Mode

Create or update .claude/config.json:

{
  "outputStyle": "data_analyst",
  "formatting": {
    "defaultFormat": "markdown",
    "tables": {
      "maxRows": 50,
      "precision": 2,
      "highlightAnomalies": true
    },
    "charts": "ascii",
    "summary": true
  }
}

Step 2: Create Analysis Skill

Create .claude/skills/analyze_data/skill.md:

# Analyze Data Skill

## Description
Analyze CSV or JSON data and return formatted summary with key insights.

## When to Use
When asked to analyze, summarize, or explore data files.

## Instructions
1. Load data file
2. Generate summary statistics (count, mean, median, std)
3. Identify outliers (IQR method)
4. Return findings in markdown table format
5. Highlight 3-5 key insights

ü§î Common Questions & Troubleshooting

Q: How do I switch output styles mid-project?

A: Use the style flag: claude --style data_analyst "your task"

Q: Can I create my own output style?

A: Yes! Define custom formatters in your project's style config. Document the format specification clearly.

Q: Does output style affect analysis quality?

A: No‚Äîanalysis quality is unchanged. Style only affects presentation format.

Q: How do I export formatted tables to Excel?

A: Markdown tables convert easily: export to HTML, then import to Excel, or use pandoc CLI for direct conversion.
