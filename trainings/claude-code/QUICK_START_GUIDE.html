Installation (5 minutes)

Prerequisites

  • Node.js 18+: Download here
  • VS Code or Terminal: Any IDE/terminal where you can run commands
  • Anthropic API Key: Get it from Anthropic Console

Installation Steps

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify installation
claude --version

For Windows Users

  • Option 1 (Recommended): Use Git Bash (requires Git for Windows)
  • Option 2: Use WSL (Windows Subsystem for Linux)

Your First 5-Minute Example

Step 1: Create a Test Project

# Create a project folder
mkdir my-claude-project
cd my-claude-project

# Initialize git (Claude Code works best with git)
git init
git config user.name "Your Name"
git config user.email "your@email.com"

# Create a simple Python file to work with
cat > hello.py << 'EOF'
def greet(name):
    print(f"Hello, {name}!")

if __name__ == "__main__":
    greet("World")
EOF

# Commit it
git add hello.py
git commit -m "Initial commit"

Step 2: Launch Claude Code

claude

You should see:

Claude Code v1.x.x
Ready for commands!

Step 3: Ask Claude to Enhance Your Code

# Inside Claude Code terminal, type:
Add type hints to @hello.py and create a function to add numbers
What happens:
  • Claude analyzes your file
  • Proposes changes
  • Asks for approval
  • Makes the changes
  • You'll see the updated file in your editor

Step 4: Review and Commit

# In Claude Code:
# Press 'Y' to approve changes shown in the approval prompt

# Then exit and commit:
exit

# Back in regular terminal:
git add hello.py
git commit -m "Add type hints and calculator function"
That's it! You've successfully used Claude Code. ğŸ‰

Core Concepts Explained Simply

1. Plan Mode (Safety First)

Press: Shift + Tab (twice)
What it does: Shows you Claude's plan BEFORE making changes
  • Review what Claude will do
  • Approve or reject the plan
  • Safe for important projects

2. File References (@syntax)

# Reference files in your prompts to give Claude context
claude "Fix the bug in @src/utils/helpers.py and also check @src/main.py"
Why: Prevents Claude from modifying wrong files

3. Memory File (CLAUDE.md)

Every project can have a CLAUDE.md file that Claude automatically reads:

# Generate one
/init

# Or create manually with project preferences:
# My Project Notes:
# - Use Python 3.11+
# - Always write tests alongside code
# - Never run the server automatically
# - Use 2-space indentation

4. Context Commands

/clear      # Clear conversation history (start fresh)
/compact    # Summarize what we've talked about
/cost       # Check API usage and costs
/mcp        # Manage MCP server connections

Practical Workflow Examples

Example 1: Simple Feature Addition

claude
# In Claude Code terminal:

Add a function called calculate_tax that takes a price (float) and returns the tax (5%)
Claude will:

1. Ask "Should I add this to hello.py?"

2. Write the function

3. Add tests (if you asked)

4. Show you the changes

Example 2: Debug an Issue

claude

# Paste error or describe issue:
The function is returning 'undefined' - please debug and fix @src/calculator.py

# Or paste actual error:
TypeError: 'NoneType' object is not iterable at line 42
Claude will:

1. Read your file

2. Identify the issue

3. Write tests to reproduce it

4. Fix it

5. Verify tests pass

Example 3: Refactor Code with Plan Review

# Press Shift+Tab twice to enable Plan Mode first

claude "Refactor @src/data_processor.py to use async/await instead of callbacks"
Claude will:

1. Show detailed plan with steps

2. Wait for your approval

3. Implement the refactoring

4. Run tests to ensure it still works

Essential Keyboard Shortcuts

Action Shortcut
Enable Plan Mode Shift + Tab (twice)
Toggle Auto-Accept Shift + Tab (once)
Take Screenshot (to show Claude) Windows: Win + Shift + S / Mac: Cmd + Shift + 4
Paste Screenshot Ctrl + V
Interrupt Claude ESC
Rewind Conversation ESC (twice)
Exit Claude Code Type exit

Common First-Time Mistakes & How to Fix Them

âŒ Mistake 1: Not Including File References

# Bad - Claude might modify the wrong file
claude "Add a new function"

# Good - Claude knows exactly where to add it
claude "Add a function to @src/utils.py"
Fix: Always use @filename when referencing specific files

âŒ Mistake 2: Forgetting to Commit Before Large Changes

# Before asking Claude for big changes, commit first:
git add -A
git commit -m "Before attempting large refactor"
Fix: Git allows you to undo mistakes easily

âŒ Mistake 3: Too Much Context at Once

# Bad - Claude gets overwhelmed
claude "Here's my entire codebase. Refactor everything to be better."

# Good - Specific, focused request
claude "In @src/auth.py, refactor the login function to handle error cases better"
Fix: Break large tasks into smaller, focused requests

âŒ Mistake 4: Not Using Plan Mode for Important Changes

# Before enabling auto-accept, always review:
Shift + Tab Shift+Tab

# Then describe your change
# Review the plan Claude shows
# Press 'Y' to approve
Fix: Use Plan Mode for anything production-related

Setting Up for Success

1. Initialize Your Project Memory

cd your-project
claude
/init
# Review and customize the generated CLAUDE.md file
exit

2. Create a .gitignore

# Make sure these are ignored:
echo ".env" >> .gitignore
echo "node_modules/" >> .gitignore
echo "__pycache__/" >> .gitignore
echo ".DS_Store" >> .gitignore
git add .gitignore
git commit -m "Add gitignore"

3. Keep Your Commits Atomic

# Good workflow:
git commit -m "feature: add login function"
claude "Enhance the login function"
git commit -m "refactor: improve login error handling"

Advanced Features to Explore Later

Once you're comfortable with basics:

MCP Servers (Extend Capabilities)

# Add Playwright for UI testing
claude mcp add playwright npx @playwright/mcp@latest

# Take automated screenshots and test UIs
claude "Using Playwright, visit http://localhost:3000 and verify the button is clickable"

Git Worktrees (Work on Multiple Features)

# Work on 3 features in parallel without conflicts
git worktree add .trees/feature1
git worktree add .trees/feature2
git worktree add .trees/feature3

# Each worktree isindependent
cd .trees/feature1 && claude "Build feature 1"
cd .trees/feature2 && claude "Build feature 2"

Custom Commands (Reuse Prompts)

mkdir -p .claude/commands
# Create .claude/commands/add-tests.md with your prompt template
# Then reuse: /add-tests "For the auth module"

How to Get Help

When Something Goes Wrong

Claude is getting confused: 
/clear          # Clear history and start fresh
/compact        # Summarize the conversation  
ESC ESC         # Rewind to an earlier point
You want to check costs: 
/cost           # See API usage
You need to troubleshoot: 
!echo $PWD      # Run shell commands (prefix with !)
!ls -la         # List files to verify what exists
!python --version  # Check Python version

Pro Tips from Experienced Users

Tip 1: Start Small, Build Big

# Don't ask Claude to build your entire app in one prompt
# Instead:
1. Build the data model
   claude "Create data.py with User and Post classes"
2. Add one API endpoint
   claude "Add /api/posts GET endpoint to app.py"
3. Add tests
   claude "Write tests for the POST endpoint"
4. Iterate and improve

Tip 2: Use Specific Language

# Vague
claude "Make the code better"

# Specific
claude "Refactor the form validation in @components/LoginForm.tsx to use a custom hook instead of inline logic"

Tip 3: Let Claude Explain Before Changing

# Ask Claude to understand first
claude "Explain the flow of data in @src/data_pipeline.py step by step"

# Then ask it to improve
claude "Now refactor @src/data_pipeline.py to be more efficient"

Tip 4: Test After Every Change

# Always ask Claude to write tests or run them
claude "Write unit tests for the new calculateTax function in @utils/tax.js"
claude "Run the test suite and show me results"
