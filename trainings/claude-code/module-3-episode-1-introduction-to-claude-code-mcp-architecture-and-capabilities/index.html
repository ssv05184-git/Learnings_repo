üîë Core Concepts Explained

Concept 1: MCP Architecture

Definition:
MCP is a standardized protocol for connecting client applications (Claude Code) to server implementations (GitHub, Figma, etc.) through request-response messages.

Why It Matters:
Standard protocols enable ecosystem growth. Anyone can build an MCP server; Claude Code can use them without modification.

When To Use:
Whenever you need to integrate external systems with Claude workflows.

Concept 2: Capabilities & Resources

Definition:
MCP servers expose capabilities (what they can do) and resources (what data they can access). Claude discovers and uses these dynamically.

Why It Matters:
Claude doesn't need pre-programmed knowledge of each server. It learns server capabilities at runtime and adapts.

When To Use:
When building servers, clearly define capabilities so Claude knows what's possible.

Concept 3: Security & Authentication

Definition:
MCP servers are isolated processes. Claude communicates through defined APIs. Credentials are managed centrally.

Why It Matters:
Isolation prevents one server compromise from affecting others. Centralized credential management reduces secrets sprawl.

When To Use:
Always‚Äîsecurity is built-in to MCP design.

MCP Protocol Basics

Request-Response Pattern:

// Client Request
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "resources/list",
  "params": {}
}

// Server Response
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "resources": [
      {
        "uri": "github://owner/repo/blob/main/README.md",
        "name": "README.md",
        "description": "Project documentation"
      }
    ]
  }
}

üí° Practical Tips & Tricks

Pro Tip 1: Server Discovery

The Technique: List available servers and their capabilities.

# List installed MCP servers
claude --list-mcp-servers

# Describe server capabilities
claude --mcp-describe github

Expected Result: Know what's available before designing workflows.

Pro Tip 2: Credential Management

The Technique: Use environment variables for server authentication.

export GITHUB_TOKEN="ghp_..."
export FIGMA_TOKEN="figd_..."
export SENTRY_AUTH_TOKEN="..."

# Servers automatically authenticate
claude "Read GitHub repository"

Expected Result: Secure, centralized credential management.

Pro Tip 3: Resource Caching

The Technique: Cache frequently-accessed resources to reduce API calls.

# In Claude config
{
  "mcp": {
    "caching": {
      "enabled": true,
      "ttl_seconds": 3600,
      "servers": ["github", "figma"]
    }
  }
}

Expected Result: Faster workflows; fewer API rate limits.

üõ† Implementation Guide

Step 1: Install MCP Servers

# GitHub
npm install @modelcontextprotocol/server-github

# Figma
npm install @modelcontextprotocol/server-figma

# Sentry
npm install @modelcontextprotocol/server-sentry

# Notion
npm install @modelcontextprotocol/server-notion

Step 2: Configure Claude Code

Create .claude/mcp.json:

{
  "servers": [
    {
      "name": "github",
      "command": "node",
      "args": ["node_modules/@modelcontextprotocol/server-github/index.js"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    {
      "name": "figma",
      "command": "node",
      "args": ["node_modules/@modelcontextprotocol/server-figma/index.js"],
      "env": {
        "FIGMA_PERSONAL_TOKEN": "${FIGMA_TOKEN}"
      }
    }
  ]
}

Step 3: Test Integration

# Start MCP servers
claude --init-mcp

# Test server availability
claude "List GitHub repositories"

ü§î Common Questions & Troubleshooting

Q: How do I check if MCP server is running?

A: claude --mcp-status shows all connected servers.

Q: What if an MCP server fails?

A: Claude automatically retries. Check .claude/logs/mcp_errors.log for details.

Q: Can I write my own MCP server?

A: Yes! See Episode 3 for building custom servers.
